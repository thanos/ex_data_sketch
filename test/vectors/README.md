# Test Vectors

This directory contains deterministic test vectors for verifying sketch algorithm
correctness and cross-backend parity.

## Naming Convention

Vector files are named: `{sketch}_{version}_{description}.bin`

## HLL Vectors

- `hll_v1_empty_p14.bin` — Empty HLL sketch, p=14 (16,388 bytes)
- `hll_v1_100items_p14.bin` — HLL after inserting "item_0".."item_99", p=14
- `hll_v1_10000items_p14.bin` — HLL after inserting "item_0".."item_9999", p=14

## CMS Vectors

- `cms_v1_empty_w2048_d5_c32.bin` — Empty CMS, width=2048, depth=5, 32-bit counters (40,969 bytes)
- `cms_v1_100items_w2048_d5_c32.bin` — CMS after inserting "item_0".."item_99"

## Theta Vectors

- `theta_v1_empty_k4096.bin` — Empty Theta sketch, k=4096 (17 bytes)
- `theta_v1_100items_k4096.bin` — Theta after inserting "item_0".."item_99", k=4096 (817 bytes, exact mode)
- `theta_v1_10000items_k4096.bin` — Theta after inserting "item_0".."item_9999", k=4096 (32,785 bytes, estimation mode)

## Generation

Vectors are generated by deterministic input sequences using `ExDataSketch.Hash.hash64/1`.
Items are strings of the form `"item_0"`, `"item_1"`, etc. The hash function uses
`:erlang.phash2/2` with Murmur3-style finalization, seeded at 0.

To regenerate:

```elixir
mix run -e '
alias ExDataSketch.{HLL, CMS}

hll_empty = HLL.new(p: 14)
File.write!("test/vectors/hll_v1_empty_p14.bin", hll_empty.state)

items_100 = for i <- 0..99, do: "item_#{i}"
hll_100 = HLL.from_enumerable(items_100, p: 14)
File.write!("test/vectors/hll_v1_100items_p14.bin", hll_100.state)

items_10000 = for i <- 0..9999, do: "item_#{i}"
hll_10000 = HLL.from_enumerable(items_10000, p: 14)
File.write!("test/vectors/hll_v1_10000items_p14.bin", hll_10000.state)

cms_empty = CMS.new(width: 2048, depth: 5, counter_width: 32)
File.write!("test/vectors/cms_v1_empty_w2048_d5_c32.bin", cms_empty.state)

cms_100 = CMS.from_enumerable(items_100, width: 2048, depth: 5, counter_width: 32)
File.write!("test/vectors/cms_v1_100items_w2048_d5_c32.bin", cms_100.state)

alias ExDataSketch.Theta

theta_empty = Theta.new(k: 4096)
File.write!("test/vectors/theta_v1_empty_k4096.bin", theta_empty.state)

theta_100 = Theta.from_enumerable(items_100, k: 4096)
File.write!("test/vectors/theta_v1_100items_k4096.bin", theta_100.state)

theta_10000 = Theta.from_enumerable(items_10000, k: 4096)
File.write!("test/vectors/theta_v1_10000items_k4096.bin", theta_10000.state)
'
```

## Versioning

Each vector file includes the sketch binary format version in its name.
When the binary layout changes, new vectors are generated with the new version
and old vectors are retained for backwards-compatibility testing.
